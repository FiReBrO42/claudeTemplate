# Harness 檔案地圖

> 本目錄是 invest 的開發 Harness（工作流防閉環迭代機制），範本 2026-07-04 由 Fable 5 bootstrap、2026-07-05 於 invest 分支重置，供 Opus 4.8（指揮官）／Sonnet 5（主力）／Haiku 4.5（廉價批次）長期運作。
> 新 session 開場先讀本檔（~1 分鐘），再依下表按需深入。

## 檔案地圖

| 檔案 | 一句話用途 | 誰最常讀 |
|---|---|---|
| `00-baseline.md` | 制度依據：基線清單、三大反模式、**能力極限（誠實條款）**、技術棧確定後的補完清單 | 所有人，開局與存疑時 |
| `01-model-dispatch.md` | 調度合約：指揮官不下場紅線、派工三件套、升降級路徑、隔離驗證 | 指揮官，每次委派前 |
| `02-judgment-matrix.md` | 三張檢核表：停損／完成／熔斷，每條附正反例 | WORKER 宣告完成前、修復失敗後；指揮官收報時 |
| `03-delegation-templates.md` | 四份填空派工模板：搜尋研究／功能實作／重構／審查 | 指揮官，寫派工 prompt 時 |
| `04-knowledge-protocol.md` | 知識分流（專屬/通用）、雙寫、播種、條目格式、**權限分級**、精簡觸發 | 所有人，踩坑落檔前；想改 harness 時 |
| `05-handover-letter.md` | 交接信：三件關鍵事、制度退化模式與預防、未竟事項 | 新 session 接手時；制度出現異常時 |
| `06-dev-workflow.md` | 開發流程閉環：任務分級（S/M/L）＋六站進出關卡，需求→交付→回顧 | 指揮官，接到開發任務時 |
| `07-stack-profile.md` | 技術棧設定檔：棧相關具體值的唯一存放處（全留白待填，附 fallback） | 所有人，需要棧相關命令/規範時 |
| `lessons/general.md` | 通用教訓（**團隊權威版**，進 git） | 所有人，開工前 |
| `lessons/project-lessons.md` | 本專案專屬踩坑檔（進 git） | 所有人，開工前 |

> 各檔標題的 A–G 字母對應 bootstrap 規格的交付清單編號；**交付項 B＝專案根 CLAUDE.md**（不在本目錄），故本目錄字母不連續並非缺檔。06/07 為 bootstrap 後新增檔案，不屬原 A–G 清單，故標題無字母。

## 目錄外的相關位置

| 位置 | 角色 |
|---|---|
| `/CLAUDE.md`（專案根） | 路由中心＋驗證命令區＋最小核心規則 |
| `/USAGE.md`（專案根） | **給人類**的使用指南＋三個初始化 prompt（日常／技術棧確定日／新專案套範本），模型無需主動讀 |
| `~/.claude/harness-global/` | 個人通用教訓累積器（不進專案 repo、自身為獨立 git repo，僅供開新專案播種；非權威版） |

## 專案 Skills（`.claude/skills/`，進 git，2026-07-05 經 User 指示建立）

| Skill | 類型 | 用途 |
|---|---|---|
| `/dev-task` | 場景 | 開發任務標準入口：站 0 分級後走 06 六站流程 |
| `/stack-setup` | 場景 | 技術棧確定日補完：填驗證命令、07 設定檔、環境設定 |
| `/harness-init` | 場景 | 範本複製後的新專案初始化：重置＋播種 |
| `/delegate` | 工程化 | 依 03 模板生成並發出派工單 |
| `/acceptance` | 工程化 | 隔離驗收（06 站 4）：派 fresh-context 驗收 |
| `/lesson-log` | 工程化 | 教訓分流落檔＋雙寫＋回顧兩問（06 站 6） |

原則：**skill 是薄路由**——只定觸發與執行順序，規則本體永遠在本目錄的 harness 檔案中；skill 內容與 harness 不一致時，以 harness 為準並修 skill。

## 一分鐘上手（給趕時間的新 session）

1. 接到任務 → 按 `03` 的模板派工，紅線見 `01` 第 2 節。
2. Subagent 說做完了 → 派 fresh-context 驗收（`01` 第 5 節），對照 `02` 第 2 表。
3. 卡住/重試 → 過 `02` 第 1 表（停損）與第 3 表（熔斷），不要硬撞。
4. 踩坑 → 按 `04` 分流當場落檔。
5. 想改本目錄任何規則檔的結構 → 先問 User（`04` 第 5 節）。
