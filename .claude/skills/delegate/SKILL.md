# 派工單生成與發派

1. **選模板**：讀 `.claude/harness/03-delegation-templates.md`，按任務型態選——模板 1（搜尋研究）／2（功能實作）／3（程式碼重構）／4（程式碼審查）。
2. **填單鐵律**（03 檔頭）：「[中文提示文字]」佔位符**全部填實**才能發出；「- [ ]」核取方塊保持未勾。實作/重構單必填【分級】欄（判定規則見 `.claude/harness/06-dev-workflow.md` 站 0）。
3. **自檢三件套**（`.claude/harness/01-model-dispatch.md` §3）：明確目標與背景／可判定真偽的驗收條件／標準回報格式（禁貼大段程式碼）。缺一重填，不發。
4. **選模型發派**（01 §1）：需要判斷的任務 → WORKER（Sonnet）；已由樣本驗證可複製的機械批次 → CHEAP（Haiku）。
5. **收報檢查**：回報不符模板「回報格式」→ 退回要求重報，不要自己動手整理；失敗處理走 01 §4 決策序列（同一件事最多兩輪）。
