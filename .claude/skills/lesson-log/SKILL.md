# 教訓落檔與回顧

1. **回顧兩問**（`.claude/harness/04-knowledge-protocol.md` §7）：這次有沒有還沒落檔的坑？有沒有哪條 harness 規則造成摩擦？——有摩擦依 04 §5 在對話中提案並引用原文，**不動規則檔**。
2. **分流**（04 §1）：內容提到本專案檔名/函式/API/業務規則/目錄結構 → 專屬（寫 `.claude/harness/lessons/project-lessons.md`）；換任何專案仍成立 → 通用（雙寫，見第 4 步）；無法判定 → 寫專屬檔＋標註〔待確認是否可上升為通用〕。**因猶豫而不寫是最大的錯。**
3. **格式**（04 §4）：六欄缺一不可——〔症狀〕〔觸發條件/重現步驟〕〔根因〕〔正確做法〕〔判準〕〔日期〕。「要小心 X」式條目不合格，等於沒寫。
4. **通用教訓雙寫**（04 §2）：先寫 `.claude/harness/lessons/general.md`（團隊權威版），再鏡射至 `~/.claude/harness-global/general.md`（條目加〔來源專案〕欄），鏡射後在該目錄執行 `git add -A && git commit`。
5. **精簡檢查**（04 §6）：落檔後看一眼——任一 lessons 檔 >40 條或 >800 行 → 向 User 提精簡案（合併哪些、上升哪些、預計刪哪些）。
